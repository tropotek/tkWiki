Introduction

This project provides HTML based web sites with the ability to publish Kooloivoo product listings. Javascript is employed to render the product details to the clients page while drawing current product data from the Koolivoo database. See Project Usage for more info.

Top

System Requirements

XML Data Server

This component was created directly for the communication between the Koolivoo MSSQL 2008 Database and the project base site. It has been tested using Windows 2003 Server, IIS V6.0 with PHP5 ISAPI Module. PHP5 uses MSSQL libs installed with PHP to connect to the Database.

Koolivoo Content Project

The content project has also been tested on the above system but will work on any system with the following installed:

  • Windows / Linux
  • IIS 6.0+ / Apache 1.2+
  • PHP5 ISAPI / PHP5 (GD, CURL)

Installation

XML Data Service

There are a number of steps required to install the XML Data server successfully. We will assume you have Windows Server 2003, MSSQL Express 2008, IIS 6.0 and PHP5 installed.

After unpacking the project archive file, there will be a folder called `dataService` this needs to be copied to a webiste folder created in IIS, this will usually reside in `C:\Inetpub` somewhere.

Then make the data directory readable and writable by IIS/apache. This will hold any files used by the service.

Next step is to get the MSSQL user details and enter them into the config.ini. There should already be some example data in the file called config.ini.in

Once completed you can use the test client that comes with the software at `http://[siteUrl]/docs/` This will give you a page outlining the Service XML Specifications and a web service javascript client that allows you to send XML queries to your service. To test the install first try an XML Ping request. Then try sending an XML request to the server.

Once the setup is successful it is recommended that you delete or close off public access to the `docs` folder.

Koolivoo Content JavaScript Server

This scomponent of the project should be able to be installed on any Windows or Linux server. From the unpacked project file we need to make the server folder accessable to IIS or Apache and make the `/data` folder writable by the web server.

The next step is to tell IIS/Apache that we want PHP to execute all .js files in the `server/djs` folder.

IIS

To do this open the `IIS Manager` navigate to the `djs` folder right click and select `Properties`. Under the `Directory` tab in the `Aplication Settings` section click the configuration button. Here we need to add a new mapping, click `Add...` and browse to your php isapi DLL, use extension `.js` and ensure `Script engine` and `Verfy file exits` are checked. That should now mak all .js files behave like php scripts.

Apache

For Apache users there is a .htaccess file included to do this for you.

Once you have successfully setup the JS Server files you need to edit the config.ini and insert the URL of where you have installed the `DataService`. Once that is done you can test the site using the files under the `client` folder. [Client] If this does not work edit the `client/index.html` file and update the javascript include line in the header of the document to reflect where the server is installed.
